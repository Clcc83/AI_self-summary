为了能够比较好的评估二元分类发的好坏，我们需要考虑4种情况：


在一个数据集检测中，会产生四类检测结果：

TP、TN 、FP 、FN：

T ——true 表示正确

F——false 表示错误

P—— positive 表示积极的，看成正例

N——negative 表示消极的，看成负例

我的理解：后面为预测结果，前面是预测结果的正确性。如：

T P—— 预测为 P （正例）, 预测对了， 本来是正样本，检测为正样本（真正例）。

T N—— 预测为 N （负例）, 预测对了， 本来是负样本，检测为负样本（真反例）。

F P—— 预测为 P （正例）, 预测错了， 本来是负样本，检测为正样本（假正例）。

F N—— 预测为 N （负例）, 预测错了， 本来是正样本，检测为负样本（假反例）。

TP+FP+TN+FN：样本总数。

TP+FN：实际正样本数。

TP+FP：预测结果为正样本的总数，包括预测正确的和错误的。

FP+TN：实际负样本数。

TN+FN：预测结果为负样本的总数，包括预测正确的和错误的

四种情形组成的混淆矩阵如下：

  • 召回率(Recall):

表示的是样本中的正例有多少被预测正确了（找得全）所有正例中被正确预测出来的比例。

用途：用于评估检测器对所有待检测目标的检测覆盖率

针对数据集中的所有正例(TP+FN)而言，模型正确判断出的正例(TP)占数据集中所有正例的比例。FN表示被模型误认为是负例但实际是正例的数据，召回率也叫查全率，以物体检测为例，我们往往把图片中的物体作为正例，此时召回率高代表着模型可以找出图片中更多的物体

  • 精确率(Precision):

表示的是预测为正的样本中有多少是真正的正样本（找得对）。预测结果中真正的正例的比例。

用途：用于评估检测器在检测成功基础上的正确率

针对模型判断出的所有正例(TP+FP)而言，其中真正例(TP)占的比例。精确率也叫查准率，还是以物体检测为例，精确率高表示模型检测出的物体中大部分确实是物体，只有少量不是物体的对象被当成物体

也就是说 recall 表示在整个检测结果中有用部分占整个数据集有用部分的比重，precision 表示在整个检测结果中有用部分占整个检测结果为有用的比重。虽然希望两个指标都能越高越好，但是这两个指标在被某些情况下存在着矛盾，所以在实际检测场景中需要根据自己的判断选择取舍或者绘制 precision-recall 曲线(简称PR曲线)帮助分析。
还有一个指标：
  • 准确率(Accuracy):模型判断正确的数据(TP+TN)占总数据的比例
缺点：准确率是分类问题中最简单也是最直观的评价指标，但存在明显的缺陷。比如，当负样本占99%时，分类器把所有样本都预测为负样本也可以获得99%的准确率。所以，当不同类别的样本比例非常不均衡时，占比大的类别往往成为影响准确率的最主要因素。

有了ConfusionMetrix以后，其实自己手工也可以计算Precision和Recall的值：

Precision = TP / (TP + FP)

Recall = TP / (TP + FN)

当一个模型有比较高的Precision和Recall的时候才可以算是比较理想的模型。Scikit-Learn也给出了计算precision和recall的函数：

Precision/Recall Trade-off

现实的情况下面是采用相同的算法时，鱼和熊掌不可兼得：当Precision提高时Recall就会下降，反之亦然。
原因是算法会利用一个Threshold的值来作为分类标准，根据一个决策函数来计算一个分数，当预测返回的评分高于threshold的时候，我们就认为分类正确，如果返回的值低于threshold的时候我们就认为分类失败。当提高threshold的值时，Precision就会提高，Recall就会下降，如下图：
上图还是对5进行分类，可以看出随着threshold的提高和下降，Precision和Recall相应的变化

2、The ROC Curve

评估单个模型的好坏基本PR评估就够用了，还有一种类似PR评估得方式叫做ROC曲线（Receiver Operation Characteristic), 这个评估有以下特点：

  1. 随机分类器是一条对角线。
  2. 完美的情况是离开这条对角线非常远。
  3. 可以用AUC指标来衡量模型离开对角线的距离。

from sklearn.metrics import roc_curve

fpr, tpr, thresholds = roc_curve(y_train_5t, y_scores)


#这里的y_scores就是通过decision function返回所有的X_train相对应的评估分数


def plot_roc_curve(x, y, label=None):

    plt.plot(x, y, linewidth=2, label=label)

    plt.plot([0,1], [0,1], 'b--')

    plt.axis([0,1,0,1])

    plt.xlabel('False Positive Rate', fontsize=16)

    plt.ylabel('True Positive Rate', fontsize=16)

plt.figure(figsize=(8,6))

plot_roc_curve(fpr, tpr)

plt.show()

而FPR（False Positive Rate）= FP /（FP + TN），即负类数据被分为正类的比例

TPR（True Positive Rate）= TP /（TP + FN），即正类数据被分为正类的比例——即recall召回率

解释：tpr True Positive Rate，越高越好，fpr False Positive Rate 越低越好。所以理想的曲线是在tpr非常接近1的时候，fpr还是非常低，完美的曲线就是y轴本身（当TP到1时，FP还是为0），所以看起来越像Y轴的曲线说明算法越好。当然正规的方式是用曲线到X轴的面积来评估，面积越接近1越好，大家可以看到中间对角线（随机分类）的面积为0.5

Tips：（PR）曲线如此相似，你可能会想知道如何决定使用哪一种曲线。
根据经验法则，当正例很少见时，或者当你更关心假正例而不是假反例时，你应该更喜欢PR曲线（非平衡数据集）。否则，请使用ROC曲线（平衡数据集）。例如，查看前面的ROC曲线（以及ROC AUC评分），您可能会认为该分类器真的很好。但这主要是因为与消极的（非5）相比，积极的（5）很少。相比之下，PR曲线清楚地表明，分类器有改进的空间（曲线可以更接近左上角）

二、Multiclass Classification

比如把输入的图片分为0，1，2，3，4，5 就是多类分类。

有些分类算法直接就支持多类分类法，比如SGD分类器，随机森林分类器和朴素的贝叶斯分类都直接支持多类分类。


有些算法只支持binary分类，比如SVM（支持向量机）但是我们也可以用多个二元分类法来做多类分类。有两种策略：

  1. One Versus the Rest（OVR）: 对10个数字分类时，我们就训练10个分类器：1或非1分类器，2或非2分类器，3或非3分类器...... 9或非9分类器。然后我们把一个待预测的值分别输入这些分类器，比如我们把一个5的图像输入，然后哪个分类器打分高，我们就认为他是哪个数字。

  2. One Versus One （OVO）：我们需要训练每一种两两比较的情况： 0 v.s. 1, 0 v.s. 2, 0 v.s. 3, 1 v.s. 2, 1 v.s. 3, 1 v.s. 4，这样10个数字，我们一共要训练 N *（N-1) / 2 = 45 个分类器！

tips：一些算法（如支持向量机分类器）适合训练集较小的。对于这些算法，OvO是首选的，因为它在小训练集上训练许多分类器比在大训练集上训练少数分类器更快。然而，对于大多数二值分类算法，OvR是首选的。Scikit-Learn检测到当你尝试对多分类任务使用二值分类算法时，它会根据算法自动运行OvR或OvO。#也可以Softmax Regression，参数设置成 multi_class="multinomial",

  1. 对每一行求和 - 因为每一行代表真实情况下面该数字有多少。
  2. 每个数字都除以该行的和，说明分配到该列（预测结果）的数字占比
  3. 用0填充对角钱，让对角线变黑。否则问题显示不出来。
完美的结果应该是一片漆黑。

row_sums = conf_mx.sum(axis=1, keepdims=True)  #对每一行的数字进行求和

norm_conf_mx = conf_mx / row_sums              #每个数字都除以行的和

np.fill_diagonal(norm_conf_mx, 0)              #用0填充对角线

plt.matshow(norm_conf_mx, cmap=plt.cm.gray)

plt.show()

三、Multilabel Classification

如果我想对一个图片返回多个结果就可以用到多标签分类。比如对余一个输入，我想知道他是不是一个大于7的数字，同时也想知道他是不是一个奇数。或一张图片上多个人的名字。
from sklearn.neighbors import KNeighborsClassifier            

y_train_large = (y_train_t >= 7) #对于大于7的标为True，否则False

y_train_odd = (y_train_t % 2 == 1) #对于奇数标为True，否则标为False

y_multilabel = np.c_[y_train_large, y_train_odd] #把上面两个列并排拼起来。


knn_clf = KNeighborsClassifier()

knn_clf.fit(X_train, y_multilabel) #用拼起来的标签来训练模型。

四、Multioutput Classification

我们将在这里讨论的最后一种分类任务被称为多输出-多类分类（或简单的多输出分类）。它只是多标签分类的泛化，其中每个标签可以是多类的（即，它可以有两个以上的可能值）。为了说明这一点，让我们建立一个可以消除图像中的噪声的系统。它将以一个嘈杂的数字图像作为输入，并且它将（希望）输出一个干净的数字图像，以一个像素强度的数组表示，就像MNIST图像一样。请注意，分类器的输出是多个标签（每个像素一个标签），每个标签可以有多个值（像素强度范围从0到255）。因此，它是一个多输出分类系统的一个例子。用KNN来训练模型：把污染过的图片作为输入，干净的图片作为标签。
